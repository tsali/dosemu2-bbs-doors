# dosemu2-bbs-doors

Run classic DOS BBS door games on modern Linux using **dosemu2** with virtual COM1.

Works with **Mystic BBS**, and adaptable to any BBS that generates DOOR.SYS drop
files. Tested with LORD (Legend of the Red Dragon) — the same pattern works for
TradeWars 2002, Barren Realms Elite, Usurper, Operation Overkill, and any DOS door
game that communicates via COM port.

This took dozens of hours of debugging to get right. Every gotcha documented below
was discovered through painful trial and error. This guide exists so you don't have
to repeat that process.

## Architecture

```
Telnet/TLS Client (SyncTERM, NetRunner, etc.)
       |
   BBS Software (Mystic, etc. — Linux, port 23)
       |
   door-wrapper.sh %N  (called by BBS with node number)
       |
   dosemu2 -E "C:\NODEx\RUNxxxx.BAT" 1>/dev/null
       |                                    |
   DOS boots (FreeDOS/FDPP)            stdout → /dev/null
       |                              (suppresses boot noise)
   RUNxxxx.BAT:
       C:
       CD \<GAMEDIR>
       CALL START.BAT <node>
       EXITEMU
       |
   GAME.EXE ←→ COM1 ←→ $_com1="virtual" ←→ PTY ←→ BBS ←→ User
```

**How it works:** dosemu2's `$_com1 = "virtual"` maps DOS COM1 to the Unix PTY
that the BBS provides. The door game reads/writes COM1 for all user I/O.
dosemu2's video output goes to `/dev/null` so users never see DOS boot messages.
All game display flows through the serial port, exactly as door games were
designed to work on a real BBS in the 1990s.

| LORD | 4.02a | Tested and confirmed working |
| Hypervisor | Synology VMM (DS720+) | Intel J4125 — no nested virt / no KVM |

**No KVM required.** This works with `$_cpu_vm = "emulated"` (pure software
emulation). If your host has KVM (`/dev/kvm` exists), dosemu2 will be faster,
and you can remove the `$_cpu_vm` lines from the dosemurc.

## Complete Directory Layout (LORD Example)

This is the exact on-disk structure required. Other door games follow the same
pattern — replace `LORD/` with the game's directory.

```
/mystic/doors/lord/                  # Or wherever you put your doors
├── lord.sh                          # Door wrapper (from this repo)
│
└── C/                               # This entire directory IS the DOS C: drive
    │                                # dosemu2 mounts it via $_hdimage
    │
    ├── LORD/                        # LORD game directory
    │   ├── LORD.EXE                 # Game binary (NOT included — obtain separately)
    │   ├── LORD.OVR                 # Game overlay file (NOT included)
    │   ├── LORD.DAT                 # Game configuration (NOT included)
    │   ├── START.BAT                # *** FROM THIS REPO *** — must have /DREW flag
    │   ├── NODE0.DAT                # Local play node config (comes with LORD)
    │   ├── NODE1.DAT                # Per-node config for node 1 (from template)
    │   ├── NODE2.DAT                # Per-node config for node 2
    │   ├── NODE3.DAT                # ...up to your max nodes
    │   ├── NODE4.DAT
    │   ├── NODE5.DAT
    │   ├── LENEMY.DAT               # Game data files (come with LORD)
    │   ├── LGAMETXT.DAT
    │   ├── LORDEXT.DAT
    │   ├── LORDTXT.DAT
    │   ├── LORDRIP.DAT
    │   ├── PLAYEXT.DAT
    │   ├── BADSAY.DAT
    │   ├── GOODSAY.DAT
    │   ├── NORMSAY.DAT
    │   ├── 3RDPARTY.DAT
    │   ├── LOCKOUT.DAT
    │   └── PLAYER.DAT               # Created automatically on first play
    │                                 # Do NOT create this as an empty file!
    │
    ├── NODE1/                       # Per-node runtime directory (node 1)
    │   ├── DOOR.SYS                 # Drop file — generated by wrapper each session
    │   └── RUNxxxx.BAT             # Temp batch — generated and cleaned up by wrapper
    ├── NODE2/                       # Per-node runtime directory (node 2)
    ├── NODE3/
    ├── NODE4/
    └── NODE5/
```

**Runtime directories** (ephemeral, recreated each run):

```
/tmp/dosemu-node1/                   # dosemu2 local config dir (node 1)
└── dosemurc                         # Generated per-run by wrapper

/tmp/lord-node1-<timestamp>.log      # dosemu2 debug log per session
```

## Quick Start (LORD)

### Step 1: Install dosemu2

```bash
sudo add-apt-repository ppa:dosemu2/ppa
sudo apt update
sudo apt install dosemu2

# Verify it works
echo "exitemu" | dosemu -dumb 2>/dev/null && echo "dosemu2 OK"

# If fdpp is not in the PPA, build from source:
# git clone https://github.com/dosemu2/fdpp.git
# cd fdpp && mkdir build && cd build && cmake .. && make && sudo make install
```

### Step 2: Obtain LORD

Download LORD 4.x (e.g., from [gameport.com](https://www.gameport.com/bbs/lord.html))
and extract to your door's `C/LORD/` directory.

### Step 3: Install Wrapper Files

```bash
DOORDIR="/mystic/doors/lord"

# Copy the wrapper
cp lord/lord.sh "$DOORDIR/lord.sh"
chmod +x "$DOORDIR/lord.sh"

# IMPORTANT: Replace LORD's original START.BAT with ours
cp lord/START.BAT "$DOORDIR/C/LORD/START.BAT"

# Create per-node configs and directories
for i in 1 2 3 4 5; do
    mkdir -p "$DOORDIR/C/NODE${i}"
    sed "s/for node 1/for node $i/;s/NODE1/NODE$i/" lord/NODE1.DAT.example \
        > "$DOORDIR/C/LORD/NODE${i}.DAT"
done

# Edit lord.sh — update DOORDIR path if not /mystic/doors/lord

# Ensure ~/.dosemu exists
mkdir -p ~/.dosemu

# Delete empty PLAYER.DAT if it exists
[ -f "$DOORDIR/C/LORD/PLAYER.DAT" ] && [ ! -s "$DOORDIR/C/LORD/PLAYER.DAT" ] && \
    rm "$DOORDIR/C/LORD/PLAYER.DAT"

# Install log rotation
sudo cp lord-log-rotate.sh /usr/local/sbin/
sudo chmod +x /usr/local/sbin/lord-log-rotate.sh
(sudo crontab -l 2>/dev/null; echo "0 2 * * 0 /usr/local/sbin/lord-log-rotate.sh") | sudo crontab -
```

### Step 4: Configure Your BBS

In Mystic's door config (`mystic -cfg`):

- **Command**: `/mystic/doors/lord/lord.sh %N`
- `%N` = Mystic's node number macro (1-5)

### Step 5: Test

Telnet into your BBS, go to doors menu, launch LORD.

To verify per-node isolation:
```bash
for i in 1 2 3 4 5; do
    echo "NODE${i}: $(grep BBSDROP /path/to/C/LORD/NODE${i}.DAT)"
done
```

## Adapting for Other Door Games

The wrapper pattern works for any DOOR.SYS-based DOS door game. To add a new
door (e.g., TradeWars 2002):

1. **Copy lord.sh** as a template (e.g., `tradewars.sh`)
2. **Change `DOORDIR`** to point to the new door's location
3. **Change the batch file command** — replace `CALL START.BAT` with whatever
   launches the game (e.g., `TW2002.EXE`)
4. **Create per-node directories** and config files if the game uses them
5. **Check the game's startup requirements** — some have their own quirks

The DOOR.SYS fixes in the wrapper (CRLF line endings, 16-bit time cap, blank
line fill) apply to virtually ALL Turbo Pascal door games from the '90s. Keep
them in your wrapper.

| Setting | LORD | TradeWars 2002 | Generic |
|---------|------|----------------|---------|
| Start command | `LORD.EXE %1 /DREW` | `TW2002.EXE` | Check game docs |
| Node config | NODE*.DAT | Game-specific | Check game docs |
| Drop file | DOOR.SYS | DOOR.SYS | Usually DOOR.SYS |
| FOSSIL needed? | No | No | No — virtual COM1 handles it |

## How the Wrapper Works (Detailed)

1. **Validates DOOR.SYS** exists in BBS temp directory (`/mystic/tempN/`)
2. **Fixes DOOR.SYS** for DOS compatibility:
   - Writes with CRLF (`\r\n`) line endings (required for DOS)
   - Caps line 18 (seconds remaining) to 32000 (prevents 16-bit overflow)
   - Caps line 19 (minutes remaining) to 500
   - Fills blank lines with "NONE" (Mystic leaves some fields empty)
3. **Copies fixed DOOR.SYS** to per-node directory (`C/NODEx/`)
4. **Generates a random batch file** (`RUNxxxx.BAT`) with CRLF endings —
   random name prevents collisions if cleanup fails
5. **Writes per-node dosemurc** to `/tmp/dosemu-nodeN/`
6. **Sets terminal environment**: `TERM=xterm`, `HOME=/home/<user>`
   (Mystic sets `TERM=dumb` and may set `HOME=/root`, both cause issues)
7. **Launches dosemu2** with `1>/dev/null` to suppress video/boot output
8. **Cleans up** temp batch file and resets terminal state

## Gotchas Reference

Every one of these was a real problem that caused hours of debugging.

### 1. LORD requires the `/DREW` flag (LORD-specific)

LORD.EXE has a compiled-in check requiring `/DREW` as a command-line argument.
Without it, LORD silently exits: `** STOP! ** You MUST run START.BAT`.
With video suppressed (`1>/dev/null`), you won't even see this message.

**Fix:** `START.BAT` includes: `LORD.EXE %1 %2 %3 %4 %5 /DREW`

### 2. DOOR.SYS MUST have CRLF line endings (all door games)

DOS expects `\r\n`. Unix `\n` only causes field misalignment — the game reads
garbage for time remaining, username, or other fields.

**Symptom:** Game loads but immediately kicks user, wrong username, or crashes.

**Fix:** Use `printf "%s\r\n"` when generating DOOR.SYS.

**Verify:** `od -c DOOR.SYS | head -3` — look for `\r \n` at line ends.

### 3. 16-bit integer overflow in DOOR.SYS (all Turbo Pascal doors)

DOOR.SYS line 18 (seconds remaining) often exceeds 32767. Turbo Pascal's
16-bit signed integer wraps this negative = game thinks user is out of time.

**Symptom:** Instant "out of time" kick.

**Fix:** Cap line 18 to 32000, line 19 to 500.

### 4. BBS sets TERM=dumb and HOME incorrectly (Mystic-specific)

Mystic sets `TERM=dumb` and sometimes `HOME=/root`. If you ever use dosemu2
terminal mode (`-t`) for debugging, S-Lang sees "dumb" and produces no output.
The `HOME` issue breaks dosemu2's wrapper script which writes to `~/.dosemu/`.

**Fix:** Force `TERM=xterm` and correct `HOME` in the wrapper.

### 5. Empty PLAYER.DAT causes fatal error (LORD-specific)

A 0-byte `PLAYER.DAT` causes: `An error occurred reading PLAYER.DAT!`

**Fix:** Delete it. LORD creates it automatically on first play.

### 6. No FOSSIL driver needed (all door games with dosemu2)

`$_com1 = "virtual"` handles COM1 at the hardware level. FOSSIL drivers
(BNU.COM, X00.SYS) can interfere. Don't load them.

### 7. Multi-node requires per-node directories (all door games)

Without separate directories, simultaneous users overwrite each other's DOOR.SYS.

**Fix:** `NODE1.DAT` → `BBSDROP C:\NODE1\`, etc. Wrapper writes to `C/NODEx/`.

### 8. NOOPEN in NODE*.DAT causes silent exit (LORD-specific)

The `NOOPEN` keyword in NODE*.DAT means "node closed." LORD exits instantly
with zero output, zero errors.

## What Does NOT Work

Tested extensively and failed — don't waste your time:

| Approach | Why It Failed |
|----------|---------------|
| DOSBox-X nullmodem TCP serial | Broken in v2026.01.02 — TCP transmit sends 0 bytes |
| DOSBox 0.74 nullmodem | Same TCP transmit bug |
| DOSBox-X `serial1=tty` | Not a valid option |
| FOSSIL drivers (BNU.COM) | Fail silently under dosemu2 |
| QEMU persistent VMs | Overcomplicated for door games |
| dosemu2 terminal mode (`-t`) | Works but shows boot noise; `TERM=dumb` issues |

**Use dosemu2 + `$_com1 = "virtual"` + `1>/dev/null`.** It's the only clean
approach, and matches the [deadbeatz/mystic_a45](https://github.com/deadbeatz/mystic_a45)
reference implementation.

## dosemu2 Configuration Reference

Generated per-node at `/tmp/dosemu-nodeN/dosemurc`:

```ini
$_cpu_vm = "emulated"        # Required without KVM; remove if /dev/kvm exists
$_cpu_vm_dpmi = "emulated"   # Same
$_com1 = "virtual"           # Maps COM1 to controlling PTY — THE KEY SETTING
$_sound = (off)              # No sound
$_speaker = "off"            # No PC speaker
$_layout = "us"              # US keyboard layout
$_rawkeyboard = (0)          # Don't grab raw keyboard
$_hdimage = "/path/to/C +1"  # Mount directory as C: drive
```

## Troubleshooting

### Door exits instantly, no output
1. Check game's startup batch for any required flags (e.g., `/DREW` for LORD)
2. Check data files don't exist as 0-byte empty files
3. Check node config for `NOOPEN` or similar "closed" keywords
4. Check the dosemu log: `cat /tmp/lord-node1-*.log | tail -20`

### "Out of time" / instant kick after loading
1. Check DOOR.SYS has CRLF line endings: `od -c NODE1/DOOR.SYS | head`
2. Check line 18 (seconds) is under 32767: `sed -n '18p' NODE1/DOOR.SYS`
3. Check line 19 (minutes) is under 32767: `sed -n '19p' NODE1/DOOR.SYS`

### Blank screen / no game output
1. Verify `$_com1 = "virtual"` is in the dosemurc
2. Verify NODE*.DAT has `COMPORT 1` (not 0, not commented out)
3. Verify DOOR.SYS line 1 is `COM1:`
4. Verify `~/.dosemu/` directory exists

### Boot noise visible to users
1. Don't use `-t` (terminal mode)
2. Redirect stdout: `1>/dev/null`

### Users see each other's data
1. Verify per-node directories exist: `ls -d C/NODE{1,2,3,4,5}/`
2. Verify each NODE*.DAT has unique `BBSDROP C:\NODEx\`
